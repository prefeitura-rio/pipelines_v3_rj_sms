# Reports

Pasta para flows de *reports*, com nomes prefixados por "Report:".

Reports são informes ou notificações com base em dados. Por exemplo, informes sobre monitoramento de usos de plataformas, estatísticas de completude de tabelas no datalake, etc.

O objetivo principal de reports é informar alguém sobre algo relacionado aos dados. Esse processo pode envolver a extração ou criação de dados novos no datalake, mas estes servem primariamente o objetivo de notificar, e podem não ser úteis para outros objetivos.
